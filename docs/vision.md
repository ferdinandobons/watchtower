# Vision: from reactive tools to proactive software

Reactive assistants begin with a user message. Proactive software begins with an event, but it should not turn every event into an interruption.

Watchtower treats proactivity as a control problem with five separate questions:

1. What happened?
2. Does the event combine with prior evidence into a useful opportunity?
3. Is the confidence high enough?
4. Is the cost of interruption acceptable now?
5. What reversible action should be proposed?

This separation creates a reusable layer between event-producing applications and capable agents.

```text
capabilities and tools: what an agent can do
proactivity runtime: when there is a reason to wake it
human policy: what it may do without confirmation
```

The coding-agent Watchtower is the first vertical demonstration. The same event, detector, policy, evidence, and feedback loop can later support research, commitments, repeated workflows, safety monitoring, and physical-world assistance.

A successful system should be judged less by how often it speaks and more by how rarely the user regrets the interruption.
